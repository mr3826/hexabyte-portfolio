# Code Review Command

Performs comprehensive code review of changes in the current workspace.

## Steps:
1. Analyze modified files using git status
2. Review code quality, style, and best practices
3. Check for potential bugs or security issues
4. Provide actionable feedback
5. Suggest improvements

## Usage:
Type `/review` to initiate code review process

## Focus Areas:
- TypeScript/React best practices
- Supabase integration patterns
- Performance considerations
- Security vulnerabilities
- Code organization and maintainability
